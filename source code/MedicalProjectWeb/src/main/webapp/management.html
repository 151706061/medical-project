杏林影像

这是一个功能界面演示平台，也嵌入了一个浏览处理dicom图像的模块，我的设想是第一代平台首先覆盖远程读片的功能，实现流程为：
客户注册登陆本网站-->从pacs或本机上传影像到网站服务器-->客户选择服务方医生（？需要讨论提供怎样的选择方式）
-->服务方医生 收到短信或邮件通知有新任务-->服务方医生登陆本网站-->服务方医生可选择自己诊断，或交给初级医生诊断
-->初级医生诊断报告需要高级医生审核 -->客户收到短信或邮件通知，诊断报告完成-->客户付费后可查看完整的诊断报告

我设想医生用户分为两个级别，高级医生的诊断报告可以直接发送给用户，初级医生的诊断报告需要得到高级医生的审核
本网站会做出个人电脑，手机和平板电脑三个版本，功能一样，但界面元素布局有变化（适应不同尺寸的屏幕）。
网站的改版是很方便的，所以未来有新功能新业务可以马上调整网站内容。
